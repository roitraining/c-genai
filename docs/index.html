Setting up

  1. Log into https://googlecloud.qwiklabs.com and navigate to the Text Generation for Applications using Generative AI Studio class.
  2. Start the Getting Started with the PaLM API lab
  3. Follow the instructor's guidance to log into Google Cloud with lab credentials

You will NOT be following the Qwiklabs instructions for this lab. Rather, follow the instructions below.

Creating a notebook instance

  1. Go to Navigation > Vertex AI > Workbench
  2. Click on the ENABLE ALL RECOMMENDED APIS button
  3. Click on the Instance tab and then on the Instances view
  4. Click CREATE NEW
  5. Enter prompt-design as the name of your instance, leave the rest of the settings at their default values, and click on CREATE

Setting up your notebook instance

  1. When your instance has been created and you see the OPEN JUPYTERLAB button, click the button to open a new tab in your browser that connects to the instances JupyterLab software
  2. On the left side of the JupyterLab interface, you will see a vertical tool palette. Click the Git icon in the palette.
  3. Click on the Clone a repository button
  4. In the modal popup, enter the following Github URL: https://github.com/GoogleCloudPlatform/generative-ai and then click Clone

Work through the intro_prompt_design notebook

  1. In the navigation pane (which shows generative-ai as the only entry), drill down to generative-ai > language > prompts
  2. Double-click on the intro_prompt_design notebook
  3. When prompted, accept the Python 3 kernel and click Select
  4. Work through the entire notebook

Tips

  • Use Shift-Enter to execute a cell and move to the next
  • Read and understand each of the documentation cells
  • Wait for a cell to complete before executing the subsequent cell
  • After you've run through cells, feel free to go back and try changing the prompts. Write your own version of a bad prompt and a good prompt for each concept and see what happens

Work through workbooks for specific use cases

  1. In the navigation area, drill down into the examples folder
  2. Work through each of the workbooks as you did with the first notebook

  3. Log into Google Cloud with the lab-generated credentials
  4. Click on the Activate Cloud Shell button

Understanding the application

  1. In a new browser tab, visit the source code page in Github
  2. Review the code in the application.

Note specifically:

  • The cloud client library being imported
  • The chat session object being created
  • How the chat session object is stored between page updates
  • How the chat history is displayed upon page update
  • The parameters used when calling the PaLM API
  • The call made to the PaLM API

Deploying and testing the application

  1. Follow the directions found in the Setup section of the repo's README.md. Perform the tasks within the Cloud Shell terminal.

Here's what these commands are doing:

  • Cloning a Git repository to your Cloud Shell machine
  • Enabling services that will be used to run your app and do Generative AI work
  • Assigning permissions so that your app can run in Cloud Run
  • Creating an Artifact Registry repo that will store your application's container image
  • Using Cloud Build to create the container image, upload it to the repo, then deploy as a Cloud Run service
  1. Go to Navigation > Cloud Run and click on the chat service entry
  2. Click on the URL to visit your application. Test it out.

Extra credit

  1. Make some changes to your app
  2. Re-run the gcloud build command to redeploy your service
  3. Try the app after the new version has been deployed

Cleanup

  1. Make sure to delete the service with the following command:
gcloud run services delete chat --region us-central1 --platform managed --quiet

  1. Got to Navigation > Vertex AI > Language

  1. If you don't get results you like, try revising your prompt and changing the Temperature
  1. Return to the top of the Language section and click on CODE CHAT
  2. Find a relatively sophisticated SQL query (from work, online, ask instructor) and have Codey explain it to you step by step
  3. Find a SQL query that can be optimized (from work, online, ask instructor) and try to have Codey suggestion optimizations. You'll find this is quite hit-or-miss.
